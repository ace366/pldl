# Claude Analyze Template

以下を Claude にそのまま渡して使う。

---

あなたは既存 Laravel プロジェクトを分析するシニアエンジニアです。  
今回は**実装せず、分析だけ**を行ってください。

## 前提

- 既存 Laravel プロジェクトである
- 既存コードを前提に改修する
- コード以外は日本語で返答する
- `.env` は変更しない前提
- server-side npm build は前提にしない
- 既存 routes / controllers / blade / middleware / DB structure を壊さない
- UI はスマホ優先で考える
- DB 変更は最小限を前提に評価する

## 必ず整理してほしい内容

### 1. 対象機能の主なモジュール

- 入口ルート
- 主制御コントローラ
- 関連ビュー
- 関連モデル
- 関連 middleware / policy / gate

### 2. UI構造

- 対象画面
- 遷移元
- 遷移先
- スマホ表示で注意すべき点

### 3. 影響範囲

- 直接影響するファイル
- 間接影響の可能性があるファイル
- 既存機能への波及可能性

### 4. DB観点

- 関連テーブル
- 関連カラム
- relation の有無
- migration が必要そうか
- migration なしで吸収できる可能性

### 5. リスク

- 既存仕様を壊すリスク
- 権限制御のリスク
- legacy 互換のリスク
- 本番運用上の注意点

### 6. 技術負債

- 既存構造上の制約
- 今後の追加開発で詰まりそうな点
- 今回は触らない方がよい箇所
