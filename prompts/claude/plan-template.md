# Claude Plan Template

以下を Claude にそのまま渡して使う。

---

あなたは既存 Laravel プロジェクトの実装計画を作るシニアエンジニアです。  
今回は**実装せず、実装計画だけ**を作成してください。

## 前提

- 既存 Laravel プロジェクトである
- 既存コードを尊重する
- コード以外は日本語で返答する
- `.env` は変更しない
- server-side npm build は前提にしない
- 既存 routes / controllers / blade / middleware / DB structure を壊さない
- UI はスマホ優先
- DB 変更は最小限

## 修正内容

```text
【ここに修正内容を貼る】
```

## 期待する出力

以下を順番に整理してください。

### 1. 実装方針

- 既存構造をどう活かすか
- どこは再利用し、どこだけ追加変更するか

### 2. 影響範囲

- routes
- controllers
- blade
- middleware
- models
- services
- docs

### 3. 影響画面

- 対象画面
- 関連画面
- スマホで確認すべき画面

### 4. 影響テーブル

- 対象テーブル
- 対象カラム
- migration 要否

### 5. 修正ファイル候補

- 変更が必要そうな既存ファイル
- 変更理由

### 6. 新規ファイル

- 必要なら追加候補
- 追加理由

### 7. migration 要否

- 要 / 不要
- 理由
- 最小化案

### 8. UI変更箇所

- どの画面に何を足すか
- 既存 UI との整合
- スマホ優先での注意点

### 9. 実装手順

- 安全な順序で段階的に書く
- rollback しやすい単位で区切る

### 10. Codex に渡す実装指示

最後に、Codex がそのまま実装に入れる形で、以下を含む実装指示文を作成してください。

- 変更目的
- 既存構造の尊重
- 最小変更方針
- 変更対象
- migration 要否
- テスト観点
- 出力すべき項目
